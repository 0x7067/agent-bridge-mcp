## ADDED Requirements

### Requirement: Antigravity provider adapter is first-class
The system SHALL expose Google Antigravity CLI as a first-class provider adapter named `antigravity` while preserving the existing provider-neutral Agent Bridge lifecycle.

#### Scenario: Listing Antigravity capabilities
- **WHEN** a caller invokes `providers_list`
- **THEN** the response includes `antigravity` alongside existing providers.
- **AND** Antigravity capability metadata includes supported modes, launch profiles, presentation actions, output cadence, readiness, and reduced-configuration metadata.

#### Scenario: Previewing Antigravity task command
- **WHEN** a caller invokes `agent_preview` with provider `antigravity`
- **THEN** the preview reports the resolved `agy` command, print-mode arguments, cwd, timeout, prompt strategy, profile diagnostics, and redacted prompt transport.

#### Scenario: Antigravity read-only modes are not overstated
- **WHEN** a caller lists or previews Antigravity for `research` or `review`
- **THEN** capability and preview metadata do not claim verified read-only filesystem enforcement unless implementation evidence proves the selected Antigravity sandbox provides it.
- **AND** prompt guidance still tells Antigravity not to edit files for non-mutating modes.

#### Scenario: Spawning Antigravity task command
- **WHEN** a caller invokes `agent_spawn` with provider `antigravity`
- **THEN** the task manager launches the Antigravity command descriptor generated by the provider adapter and preserves the public Agent Bridge lifecycle response shape.

#### Scenario: Antigravity model option
- **WHEN** a caller supplies `model` for provider `antigravity`
- **THEN** the command descriptor passes the selected model through Antigravity's supported model flag.

#### Scenario: Antigravity binary override
- **WHEN** `AGY_BIN` is set
- **THEN** Antigravity version checks, smoke checks, previews, and task launches use that command path.

#### Scenario: Doctor reports Antigravity binary override
- **WHEN** `AGY_BIN` is present in the MCP server environment
- **THEN** doctor environment diagnostics include `AGY_BIN` without exposing secret values.
